📄 Phase 4 최종 프로젝트 요구사항 (PDF 전문 정리)

1. 프로젝트 개요 및 목표

제목: COMP322 Term Project - Phase 4 (Database Website Construction)

배경: 이전 단계(Phase 3)에서는 JDBC(Java DataBase Connectivity)를 이용해 
데이터베이스에 접근하고 조작하는 콘솔 기반의 간단한 Application을 구축했음.

목표: 이번 마지막 단계는 이전 단계에서 구축한 Application의 주요한 비즈니스 로직을 활용하여 
설계한 DB를 Back-end에서 활용하는 웹 사이트를 구축하고 서비스하는 것을 목표로 함.


2. Requirements (요구사항) [70 pts]

A. 웹 사이트 구현 [50 pts]
Phase 3에서 구현한 JDBC Application의 필수 기능을 탑재한 웹 사이트를 구현해야 함.
Phase 2에서 구성한 DB와 연동될 수 있어야 함.
웹 사이트 디자인, 프로그래밍 언어 선택 및 구축 환경은 제한 없음.
JSP 이외에 Django, Flask, Node.js 등 팀에서 자신 있는 웹 프레임워크 기술을 적용해도 됨.
단, 이 경우 Phase 3의 JDBC 프로그램을 통해 구현한 DB 조작(삽입/삭제/변경/검색) 기능과 
동등한(equivalent) 기능을 해당 프레임워크에 구현해야 함 (e.g., Python-Oracle 등).

B. Git 활용 및 평가
Github 등을 활용한 코드 관리 Repository 사용.
사용 기록, 즉 커밋 로그를 토대로 Github 활용도를 평가함.
Repository link와 Contributors Graph를 TeamX-task1.txt에 기술해야 함.

C. 사용자 동시성 제어 (Concurrency Control) [20 pts]
트랜잭션을 활용한 사용자 동시성 제어(concurrency control)를 지원해야 함.
여러 명의 사용자가 동시에 접속할 수 있도록 해야 함.
동시성 제어 실패 시, 데이터베이스 일관성, 무결성 오류가 생길 수 있음에 주의해야 함.
어떻게 해결하였는지 TeamX-Additional_task1.txt에 기술해야 함.
